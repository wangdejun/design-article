你的App中有这10个错误吗？

无论你正在打算或已经设计了一个App,或者已经创造过多个不同种类的App,你应该知道设计失误是不能容忍的，这对App的易用性和接受度是致命打击。无论何时，用户从来都选有趣的，优美的，能无脑使用的App。

但是，设计师自己已经深入进去了，有时候忽略了这些失误，因为他们一开始就知道怎么使用。现在我们就来盘点下那些App设计中容易犯的错误。

(记得下面的设计样例都是遵循了这些设计原则的，可以养养眼了)

错误1：第一观感差评

毋庸赘言，设计精良的app给人第一感觉一定是非常好的，但是如果事先下载页或者打开App的首屏就让用户不爽，他们会避之不及。第一印象跟见女友一样重要！

首页应该加载迅捷，用户能一下明白用这个App能做什么，要怎样做;最重要的是--让用户知道为什么要下载这个App而不是其他App。

需要交互的元素要已与发现和操作，并且被简洁有序地安排在一个容器，同时能够反映出来你的品牌定位和App整体的视觉风格。

错误2.没有操作暗示

你的App是用来做什么的？你通过怎样的方式来把这些传达给你的用户？

    需要怎样操作？点击屏幕哪里？点击之后的预期是什么？设计需要帮助用户准确理解交互动作。谈到把这些暗示视觉化的时候，我们需要回归到设计的基本：
  1. 色彩：如果是可点击元素，高饱和度明艳的色彩能抓人眼球。
  2. 空间：关键元素（控件）周围要留够空间，让用户能够一眼看到，容易操作。
  3. 字体：使用简洁的字体，统一的描边宽度，适合屏幕展示的文字大小，所有提示性的信息不能晦涩难懂。

错误3.凌乱无序

对很多App来说，混乱是最大的问题，似乎设计师想把一个App上每个单独的设计都堆到一屏上。

无论是游戏也好，工具应用也好，信息流App也好，一定要把这种想法丢掉!

App是在一个非常之小的屏幕上设计的，一次塞太多东西会使用户望而却步。

Think of each “screen” as a single container for a single element. When you move to the next screen, a new bit of information can be displayed. Users will swipe and scroll if the content is engaging. And they are even more likely to do so if the design is clean and easily accessible. 试着把"一屏"想象成只存放一个元素的容器，你划到下一屏，新的信息展示出来，

错误4.触控元素过小或者太近

不是每个用户都有纤纤玉指，可触控的如果太小太密集而且不能放大的话，对他们来说无亚于灾难。

保证每个元素周边有足够的空间可以减少用户误操作。空间布局没有黑魔法，但是当你错误的时候你就会知道了。(看一眼你的数据分析就可以了)。

Here’s how I created my scale. I borrowed a friend with large hands. Drew a circle around his forefinger and that’s my guideline. 我是这样确定我的触控尺寸大小的，我找来一个朋友，他有一双巨大的手，在他的食指周围画一个圈，这就是我的指南 The button doesn’t have to be that size, but the tappable area should be. 可能Button没必要，但是可触控热区必须有那么大，不能更小了。 And nothing should be any closer.

Because of this, cards and container elements to click for actions are nice. 正因为如此，元素在可触控页卡容器内对交互来说无疑是一个好的设计。 This means that anywhere you tap in an area related to the content all take the user to the same location. (It’s design friendly and user friendly.) 这意味着，无论你点击了热区的任何一个地方，都会带用户去你指定的地方，不必担心误操作 (这对设计师友好，对用户也是有好的)

错误5：不标准的图标或动作提示

There’s no reason to recreate iconography for your projects. The Facebook link is supposed to be an “f” inside a button; Twitter should be represented by a bird. Why? Because every user know exactly what these elements are and what they do. 没道理自己造一套图标UI的轮子。facebook的链接是一个带有"f"的按钮，推特是一个有小鸟的图标。因为每个人都知道这些icon是什么意思。既然这样，你为何要改变呢？

Incorporate standard iconography and actionable cues in the design. It will save you time and will help prevent user frustration. 这能节省时间，减少困惑

错误6. 设计不连贯一致，缺乏整体性，

一个App内的UI和功能都应该是高度统一的。当用户由小白渐成重度用户之后，他（她）会对App的运行和机制有稳定的预期。

As they learn the app, it needs to meet those expectations aesthetically and operationally. This includes everything from color and type choices to user tools and how they work.

错误7：不闭合的反馈循环

Every interaction should be met with an equal and opposite reaction. (It’s one of the laws of digital physics, right?)

Feedback gives the user a sense of control with engaging with an app and it provides you with valuable information about how users engage. Feedback also helps users make decisions and makes the interface easy.

The feedback loop should:

Tell users where they are or show their current state. 告诉用户他们处于什么地方，什么状态。比如鼠标Hover时候，颜色会改变。 (Example: Hover state color changes.)

Respond when an action is executed 对用户动作及时反馈，不论错误提示还是成功反馈。比如一个表单提交之后的消息提示。 (this includes success and failure). (Example: Success notification when a form is submitted.)

Tell users what is happening or what should happen next. (Example: Navigation arrows or a loading progress swheel.) 告诉用户正在做什么，接下来应该做什么，比如一个箭头提示或者一个正在下载中的进度条。

错误8:App跟所搭载平台不同步

Are you an iPhone or Android user? Ask this question in a crowded room and you will get plenty of impassioned responses. Remember this when you design and release an app.

It should be in sync with the aesthetics of the platform where it is downloaded. That might even include two slightly different versions of the app.

Android and iOS guidelines will help you do this pretty easily. But you have to take the full interface into consideration. Just think about that lonely skeuomorphic app icon on your iPhone home screen that begs to be deleted. Say what you want about the new Instagram logo, for example, but it actually matches the interface now.

错误9：交互界面太复杂

复杂是游戏App的一个通病--他们就是太TM繁琐了。当你知道要怎么操作的时候，你也不想再继续玩了。

The interface should be simple. 交互界面应当简洁，是可以浏览，应该是易于理解的， It should be scannable. It should be easy to understand and operate without an owner’s manual.

Anything else is just too complicated. 其他的内容太复杂

For most users, and app is downloaded to solve a problem. 对大多数用户来说，下载一个app一般是为了解决一个问题。 Each app is one solution. (You don’t use the same app as an alarm and to play a game, do you?) 一个app就是一个解决方案，你不会专门下载一个app来提醒你玩游戏吧，你真的会？ Remember that when you are in the design phase. What is the problem you are trying to solve? Solve it and move on. 再设计阶段，你就应该时刻记住你要解决的问题是什么，解决它，继续！

错误10： Forgetting to Set Defaults忘记默认设置

Default values in forms and interactive elements can be a big help for users. 表格和交互元素的默认值对用户来说非常有帮助。 Consider them to be a small-scale visual walkthrough. 把他们看作小的视觉指引 Default values can include anything from a pre-marked preference (most-common choice or a null option) in lists and check boxes, instructional text in entry fields or any other example illustrating what is supposed to happen.

Defaults are helpful because they can make things easier for the user by showing them how to use the app, 默认值 providing an option for users who are uncertain (as in the case with a null default) and by speeding up the entire interaction.

Defaults are also a good bit of feedback for you as well, particularly for forms, because they help cut down on the number of errors and with a “null” option, you immediately know the user did not make a selection.
