你的App中有这10个错误吗？

无论你正在打算或已经设计了一个App,或者已经创造过多个不同种类的App,你应该知道设计失误是不能容忍的，这对App的易用性和接受度是致命打击。无论何时，用户从来都选有趣的，优美的，能无脑使用的App。

但是，设计师自己已经深入进去了，有时候忽略了这些失误，因为他们一开始就知道怎么使用。现在我们就来盘点下那些App设计中容易犯的错误。

(记得下面的设计样例都是遵循了这些设计原则的，可以养养眼了)

错误1：第一观感差评

毋庸赘言，设计精良的app给人第一感觉一定是非常好的，但是如果事先下载页或者打开App的首屏就让用户不爽，他们会避之不及。第一印象跟见女友一样重要！

首页应该加载迅捷，用户能一下明白用这个App能做什么，要怎样做;最重要的是--让用户知道为什么要下载这个App而不是其他App。

需要交互的元素要已与发现和操作，并且被简洁有序地安排在一个容器，同时能够反映出来你的品牌定位和App整体的视觉风格。

错误2.没有操作暗示

你的App是用来做什么的？你通过怎样的方式来把这些传达给你的用户？

    需要怎样操作？点击屏幕哪里？点击之后的预期是什么？设计需要帮助用户准确理解交互动作。谈到把这些暗示视觉化的时候，我们需要回归到设计的基本：
  1. 色彩：如果是可点击元素，高饱和度明艳的色彩能抓人眼球。
  2. 空间：关键元素（控件）周围要留够空间，让用户能够一眼看到，容易操作。
  3. 字体：使用简洁的字体，统一的描边宽度，适合屏幕展示的文字大小，所有提示性的信息不能晦涩难懂。

错误3.凌乱无序

对很多App来说，混乱是最大的问题，似乎设计师想把一个App上每个单独的设计都堆到一屏上。

无论是游戏也好，工具应用也好，信息流App也好，一定要把这种想法丢掉!

App是在一个非常之小的屏幕上设计的，一次塞太多东西会使用户望而却步。

试着把每屏当成一个元素的容器，你通过左右上下划到下一屏，新的信息展示出来。如果设计清晰整洁的话，好的内容会带着用户不断深入。

错误4.触控元素过小或者太近

不是每个用户都有纤纤玉指，可触控的如果太小太密集而且不能放大的话，对他们来说无亚于灾难。

保证每个元素周边有足够的空间可以减少用户误操作。空间布局没有黑魔法，但是当你错误的时候你就会知道了。(看一眼你的数据分析就可以了)。

我是这样确定我的触控件尺寸大小的：我找一个有一双巨手的朋友，在他的食指周围画一个圈，这就是我的圭臬。 可能视觉上按钮没必要这么大，但是可触控热区必须以此为指南，不能再小了。

正因为如此，元素在可触控页卡容器内对交互来说无疑是一个好的设计。这意味着，无论你点击了热区的任何一个地方，都会带用户去你指定的地方，不必担心误操作 (这对设计师友好，对用户也是有好的)

错误5：不标准的图标或动作提示

没道理自己造一套图标UI的轮子。facebook的链接是一个带有"f"的按钮，推特是一个有小鸟的图标。因为每个人都知道这些icon是什么意思。既然这样，你为何要改变呢？

请在设计中尝试拥抱标准化icon和动作提示UI，这能节省时间，减少给用户带来的困惑。

错误6. 设计不连贯一致，缺乏整体性，

一个App内的UI和功能都应该是高度统一的。当用户由小白渐成重度用户之后，他（她）会对App的运行和机制有稳定的预期,包括app用色和工具选型这种细节

错误7：不恰当的反馈回路

每个交互动作要么有一致的反馈，要么有相反的反馈。反馈能给用户以掌控整个app的感觉，也会让你知道用户是怎样使用app的。反馈还能帮助用户快速做决定，让app更加易用。

给用户的反馈回路应该这样：

告诉用户现在在哪儿，处于何种状态。（比如触控按钮时，颜色会改变。）

当用户执行一个动作后，不论是报错还是已经成功，都要及时反馈。比如填写一个表单提交之后的成功消息提示。

告诉用户正在做什么，接下来应该做什么，比如一个箭头提示或者一个正在下载中的进度条。

错误8:App跟所搭载平台不同步

如果你在人群密集的一个屋子问：你是安卓用户还是iphone用户？你可能获得一群拥趸的热烈响应，你会发现这两种用户群体是不一样的，所以你的app在审美风格上需要跟app分发平台同步，有时候两个轻微差异的app版本也是必要的。

Android and iOS guidelines will help you do this pretty easily. But you have to take the full interface into consideration. Just think about that lonely skeuomorphic app icon on your iPhone home screen that begs to be deleted. Say what you want about the new Instagram logo, for example, but it actually matches the interface now.

错误9：交互界面太复杂

复杂是游戏App的一个通病--他们就是太TM繁琐了。当你知道要怎么操作的时候，你也不想再继续玩了。

The interface should be simple. 交互界面应当简洁，是可以浏览，应该是易于理解的， It should be scannable. It should be easy to understand and operate without an owner’s manual.

Anything else is just too complicated. 其他的内容太复杂

For most users, and app is downloaded to solve a problem. 对大多数用户来说，下载一个app一般是为了解决一个问题。 Each app is one solution. (You don’t use the same app as an alarm and to play a game, do you?) 一个app就是一个解决方案，你不会专门下载一个app来提醒你玩游戏吧，你真的会？ Remember that when you are in the design phase. What is the problem you are trying to solve? Solve it and move on. 再设计阶段，你就应该时刻记住你要解决的问题是什么，解决它，继续！

错误10： Forgetting to Set Defaults忘记默认设置

Default values in forms and interactive elements can be a big help for users. 表格和交互元素的默认值对用户来说非常有帮助。 Consider them to be a small-scale visual walkthrough. 把他们看作小的视觉指引 Default values can include anything from a pre-marked preference (most-common choice or a null option) in lists and check boxes, instructional text in entry fields or any other example illustrating what is supposed to happen.

Defaults are helpful because they can make things easier for the user by showing them how to use the app, 默认值 providing an option for users who are uncertain (as in the case with a null default) and by speeding up the entire interaction.

Defaults are also a good bit of feedback for you as well, particularly for forms, because they help cut down on the number of errors and with a “null” option, you immediately know the user did not make a selection.
